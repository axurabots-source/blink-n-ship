Authentication

All API endpoints require authentication using the X-API-KEY header with your
unique API key.

Header Name                      Value                Required
X-API-KEY                        Your unique API key  Yes
Content-Type                     application/json     Yes (for POST)

Important Notes

 v Security: Keep your API key secure and never expose it in client-side code or

            public repositories.

 q Content-Type: All POST requests must include Content-Type: application/json

            header.

 s Rate Limiting: Maximum 100 requests per minute per API key.
  Validation: Always validate response status before processing the data.
 s Support: For support, contact: support@flaship.pk

skipped.

Endpoint:

    https://partners.flaship.pk/mr/api/bulk/booking/

Request Parameters:

© Flaship - partners.flaship.pk  Page 10
Flaship API Documentation

Parameter                        Type    Required      Description

courierCompany                   string  Yes           Courier company name
pickuplocation                   string
                                         Yes           Pickup location ID

file                             file    Yes           CSV file (multipart/form-data)

Ways to Call:

     1. cURL multipart:
     curl -X POST .../mr/api/bulk/booking/ -H "X-API-KEY: ..." \

         -F courierCompany=Leopard -F pickuplocation=17 -F file=@orders.csv
     2. Postman: Body > form-data (courierCompany, pickuplocation, file)

Sample Response:

     {
         "success": { "...": "courier bulk booking response" },
         "error": [
            { "row_number": 3, "error": "Invalid destination City" }
         ]
